# Planning: STW-0001 — Build www.studytrax.com one-shot static site

## Approach

Delegate the entire build to a **fresh `ui-developer` agent** with the full spec inline. The agent produces all files in a single run (per the "one-shot generation" directive in spec §0). After the agent returns, run a verification pass: file-tree check, forbidden-word grep, verbatim-text grep, and validation-block-length comparison. Any failures flip the ticket back to in-progress with an `issues.md` report.

## Risks

- **Forbidden-word violation.** The agent might reach for an intuitive word (e.g., "broken", "can't") unaware it's banned. Mitigation: explicit forbidden-word list in agent prompt; automated grep verification after return.
- **Validation block shorter than ceilings block.** Agent writes ceilings block naturally, validation block as afterthought. Mitigation: explicit length requirement in prompt; char-count verification.
- **Verbatim copy drift.** Agent paraphrases instead of rendering verbatim. Mitigation: explicit "RENDER VERBATIM" markers in prompt; string-match verification.
- **Invented facts.** Agent fabricates testimonials, logos, case studies. Mitigation: explicit `[MISSING: ...]` rule in prompt; grep for common fabrication patterns.
- **Context window pressure.** Full spec + output may exceed agent's comfortable working set. Mitigation: ui-developer runs with fresh context; spec is ~10KB, output is ~150KB — well within limits.
- **Non-git repo.** The project directory is not a git repository; no commit step will run.
